# Gatsbyでブログを作ってみました

ここ二週間くらいReactにどハマりしてるので[Gatsby](https://www.gatsbyjs.com/)を使ってブログを作ってみました。

正直、GatsbyはあまりReactを書いているという
気持ちになれない(プラグインの設定の方が大変だったりする。)のですが、
今までサーバーサイドをやることばかりだったのでSEOとか色々勉強できてためになりました。

## Gatsbyを選んだ理由

前々からブログを作りたいと思っていたのですが、一応エンジニアの端くれとしては
自分で手作りしたブログの方がいいだろうとこだわりを持ってしまったために
ブログを始める心理的コストが大幅に上がってしまっていました。

そんななか、[Material-UI](https://material-ui.com/)の便利さが身に染みて
これなら僕も簡単にブログ作れるんじゃないか？と思っていたところにGatsbyの存在を
知って作ってみようと思いました。

GatsbyとNext.jsで悩みましたが、テンプレートの豊富さや、チュートリアルがブログ作り
になっていることから始めるにあたってのコストが低そうだと思いGatsbyを選びました。(この決断が正しかったのか怪しい、実際テンプレート使ってないし、、、)

## ブログに持たせたかった機能

ほしかった機能として、

- マークダウンで簡単に書ける
- ブログ内でJavaScriptのコードを実行できる
- レイアウトを柔軟に変更できる
- できるだけReactから逸脱しない(Material-UI使わないとダサい画面しか作れないので、、、)

あたりの機能が欲しいと思っていたのですが、MDXというマークダウンの拡張を使えば
比較的簡単に作れることがわかったのでそれで作ってみました。

## MDXはすごいですね

```javascript
import Button from '@material-ui/core/Button';

<Button variant="outlined" color="primary">
    Primary
</Button>
<Button variant="contained" color="secondary">
    Secondary
</Button>
```
試しにこんなコードを書いてみます。(この文字の真下に上と同じコードが書かれています！！)

import Button from '@material-ui/core/Button';

<Button variant="outlined" color="primary">
    Primary
</Button>
<Button variant="contained" color="secondary">
    Secondary
</Button>

今風のボタンがこんなに簡単に作れる凄い!!便利!!!
**Material-UI**最高!!

## 難しかった点

Reactに毛が生えたような知識で大丈夫かなと思ってたのですが、
なかなかGatsby特有の知識が必要で大変でした。

`gats-by-config.js`とか`gatsby-node.js`っていうプラグインの設定弄ってる時間の方が
ブログの画面作ってる時間より圧倒的に長かったです。

Reactと比べても仕方ないとは思うのですが、ドキュメントが少ないのがつらいですね。

あと、一番嫌なのは開発サーバを使うときにキャッシュファイルが悪さをして、
さっきは動いてたのに今は動かねえとかなりがちなのがつらいですね。

## 今後改良したい点

まず、indexのページがめちゃくちゃ雑なので丁寧に作り直したいです。
他にもタグ機能とか検索機能(検索機能は必要ないか？)とかetc...作りたいなと思っています。

一応、技術ブログにしようかなと思っているのでもう少しMDXを柔軟に書けるようにファイル構成とか少し直した方がいいかなと思っています。

もう少しGatsbyに慣れたらブログの作り方についての解説とか自分なりにしたいなとか思っています。
(公式のチュートリアルもかなり丁寧なんだけども、Reactにある程度慣れてる身からすると説明して欲しいのはそこじゃない感があったので)
